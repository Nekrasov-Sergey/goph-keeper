# 🔐 Goph Keeper

**Goph Keeper** - сервис для безопасного хранения приватной
информации.

Поддерживаемые типы секретов:

- 🔑 Логин и пароль
- 📝 Текстовые данные
- 📁 Бинарные файлы
- 💳 Банковские карты

Сервис состоит из:

- **gRPC сервера**
- **CLI клиента**

------------------------------------------------------------------------

# 🚀 Быстрый старт

## 1. Клонировать репозиторий

``` bash
git clone https://github.com/Nekrasov-Sergey/goph-keeper.git
cd goph-keeper
```

------------------------------------------------------------------------

## 2. Настроить переменные окружения

Скопируйте файл `.env.example`:

``` bash
cp .env.example .env
```

Откройте `.env` и замените значения `change_me` на реальные.

------------------------------------------------------------------------

## 3. Настроить конфигурацию

Скопируйте конфигурационный файл:

``` bash
cp config_example.yml local.yml
```

Откройте `local.yml` и замените значения `change_me` на нужные
параметры.

------------------------------------------------------------------------

# ⚙️ Сборка проекта

Первичная подготовка проекта:

``` bash
make first-build
```

Команда выполнит:

- генерацию protobuf и gRPC кода
- генерацию TLS сертификатов
- сборку сервера
- сборку клиента

------------------------------------------------------------------------

# ▶️ Запуск

## Запуск сервера

``` bash
make server
```

## Запуск CLI клиента

``` bash
make client
```

------------------------------------------------------------------------

# 🧰 Доступные команды Makefile

------------------------------------------------------------------------

`make first-run` Подготовка проекта (генерация кода и сборка)

`make server` Сборка и запуск сервера

`make client` Сборка и запуск клиента

`make build-all-client` Сборка клиента для всех платформ

`make gen` Генерация protobuf и сертификатов

`make fmt` Форматирование кода

------------------------------------------------------------------------

# 🖥 Поддерживаемые платформы клиента

CLI клиент собирается для:

- macOS (arm64)
- Linux (amd64)
- Windows (amd64)

Собранные бинарные файлы находятся в директории **build/**

------------------------------------------------------------------------

# 🔒 Безопасность

Проект использует несколько уровней защиты данных:

- **TLS** — защита gRPC соединения
- **AES** — шифрование хранимых секретов
- **bcrypt** — безопасное хэширование пользовательских паролей
- **JWT** — аутентификация и авторизация клиента

------------------------------------------------------------------------
